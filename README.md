# In (Medical) RAG We Trust?
### Evaluating Meta-Prompting Against Data Poisoning in Local SLMs

**NLP Exam Project** · Master in Data Science for Economics · Università degli Studi di Milano  
*Prof. Alfio Ferrara — Project P5: Knowledge, Retrieval & Robustness*

---

## What this is

A controlled experiment testing whether a single prompt change can make a small local language model treat its retrieved sources more critically when they contain conflicting medical claims.

**Research question:** can a meta-prompt push a 1.5B-parameter instruction-tuned model from passively summarising retrieved context toward actively flagging contradictions?

**Short answer:** it makes the model more *cautious*, not more *accurate*. The epistemic verification prompt raises the conflict-detection rate from 0% to 14%, while factual accuracy stays flat (0.41 → 0.40). The gain is caution, not correctness.

See [`paper.pdf`](./paper.pdf) for the full write-up.

## Key results

| Metric | Baseline | Epistemic Verification | Shift |
|---|---|---|---|
| Factual accuracy | 0.41 | 0.40 | −0.01 |
| Poison adoption | 0.59 | 0.46 | −0.13 |
| Flag rate | 0.00 | 0.14 | **+0.14** |
| Conflict-detection rate | 0.00 | 0.11 | **+0.11** |
| Parse-failure rate | 0.00 | 0.00 | 0 |
| Retrieval recall (pair) | 1.00 | 1.00 | 0 |

*N = 100 entities × 2 prompts = 200 greedy generations. Decoding is deterministic; numbers are exact.*

## Setup

**Model:** `Qwen/Qwen2.5-1.5B-Instruct` (local, Apple M1, 8 GB RAM, fp16)  
**Embeddings:** `sentence-transformers/all-MiniLM-L6-v2` (CPU)  
**Vector store:** ChromaDB, cosine similarity, `top_k=2`  
**Decoding:** greedy (`do_sample=False`), `seed=42`

**Dataset:** 100 fully fictional medical entities (invented drugs and diseases), each paired with one truthful and one contradictory (~80-word) document — 200 documents total. Generated by Google Gemini Pro 3.1 from a single fixed prompt; validated automatically before use. Every query forces a strict 1:1 conflict.

**Why fictional names?** Real entity names would let the model fall back on memorised pretraining knowledge. Fictional names block that escape hatch: the retrieved context is the model's *only* source of information, which isolates the effect of the prompt cleanly.

## Reproduce

```bash
# 1. Create and activate a virtual environment
python3 -m venv .venv && source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Quick smoke test (first 5 entities only)
python3 -m src.experiment --smoke

# 4. Full experiment (100 entities × 2 prompts, takes several minutes on M1)
python3 -m src.experiment
```

Results are saved to `results/experiment_records.json` and `results/experiment_metrics.csv`.  

## View the demo notebook

The notebook in `notebooks/demo.ipynb` renders all tables and figures from the `results/` directory. It does not load the model — you can run it independently of the full experiment.
